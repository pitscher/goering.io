Grandma Bot

  • Twilio API
  • Docker
  • Git

The Grandma Bot allows you to send randomly selected messages as a SMS via the Twilio API.

Some time ago, my grandmother had reminded me once again that I should contact her from time to time with a short message. She wanted to know if I was doing well. I appreciate her very much, but in the past I had always forgotten to contact her.

Because of this, the Grandma Bot was created as a weekend project. I wanted to automate the process of sending a regular "everything is okay here" message. From a list of predefined text messages, a random message is selected and sent as a SMS. Super simple.

Here is a brief overview of the features:

  • Bot sends a message and exits if done
  • Mode for checking all messages provided in a txt file can be be activated
  • Messages created by the user are checked for validity/suitability so that they can be sent as a SMS
  • Using the Twilio SMS API while handling errors that could arise
  • Well comprehensible procedure due to understandable outputs on the console
  • Operating options: native, Docker/k8s, serverless
  • Regular execution (e.g. weekly) with CRON or timed cloud functions can be implemented easily

The tool works without any problems. However, my ethical principles ultimately prevented me from actually using it. When I introduced my grandmother to the Grandma Bot, she was amazed to see what can be done with a computer these days. At the same time, she pointed out that a bouquet of flowers could not have been big enough as an excuse (if I would really use the bot).

In the end, I have build up my knowledge of JavaScript through this project. I was able to cause a smile or two among my family and friends while introducing them to what I have built.

Colloquium Selector Robot

Private
  • DigitalOcean
  • Kubernetes
  • React
  • NodeJS API
  • MariaDB
  • Bash scripting

The Colloquium Selector Robot is a tool for scheduling and coordinating presentations with a large number of participants.

The project is tailored to a concrete use case of my division at Deutsche Telekom. On one day of a month, a so-called colloquium took place.. Employees and working students came together to give presentations on various topics or to attend those of others.

The Colloquium Selector Robot will coordinate a date with all participants to host a colloquium. Users can enter their attendance and the topic of their presentation via a web interface. They can also enter the details of upcoming talks like the name of the presenter and the type/length of the talk. Queries to schedule the next session are automatically sent by mail.

The project consists of a React-based webapp, a NodeJS API, and a MariaDB database.

Some of my key activities during the project:

  • Project management activities and lead a fellow student as a software developer
  • Requirements management and planning of (further) development tasks using a Kanban board
  • Created Kubernetes manifests for running the tool in my own Kubernetes cluster

The project was my entrance into the world of Kubernetes. I was responsible for ensuring a safe operation of the tool. Therefore, I already made sure that my fellow student and I follow best practices during the development phase.
